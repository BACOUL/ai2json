Why it helps

A small contract between the web and your AI stack.

AI2JSON does one thing: turn messy HTML into clean, predictable inputs.

Less noise

Remove boilerplate (nav, footers, banners) so models focus on content.

Better chunks

sections[] are natural units for prompting, RAG indexing, and change tracking.

Deterministic

Stable JSON makes debugging and reproducibility easier than raw HTML scraping.

Hash for diffing

A SHA-256 fingerprint lets you detect meaningful content changes over time.

JSON contract

Minimal schema, designed for LLM consumption.

Keep it small: short field names reduce tokens. The main payload is always sections[].t.

Field Type Meaning
specstringAlways ai2json.v0.
urlstringRequested URL.
titlestringPage title.
langstringBest-guess language code.
hashstringSHA-256 over extracted text (prefixed with sha256:).
sections[]arrayOrdered content blocks.
sections[].hstring/nullHeading (if any).
sections[].lnumberHeading level (0–3).
sections[].tstringClean text for this section.

Public sandbox endpoint (example): GET /parse?url=<ENCODED_URL>

Sandbox limits

Free by default, protected against abuse.

This is a public sandbox. Requests are rate-limited and restricted to public HTTP(S) URLs. Very large pages and slow responses are rejected.

Responses (JSON) 200 OK 400 invalid_url 403 forbidden_url (private IPs / non-http(s)) 408 timeout 413 too_large 422 not_extractable 429 rate_limited 5xx upstream_or_internal Tip For LLM prompts, send only: title + sections[].t
