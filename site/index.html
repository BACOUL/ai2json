How AI2JSON works

AI2JSON acts as a thin, stateless bridge between the messy HTML of the public web and a stable JSON contract designed specifically for AI workloads.

1. Fetch & detect

AI2JSON fetches the target URL, follows redirects when necessary and detects the main content area using a reader-style heuristic.

2. Clean & normalize

Navigation, ads, scripts and layout noise are removed. The remaining text is normalized to a consistent structure with predictable spacing.

3. Structure into sections

Headings and paragraphs are grouped into sections[], each with a heading, level, id and clean text field that LLMs can consume directly.

4. Summarize & fingerprint

A short factual summary is generated and a SHA-256 hash is computed over the extracted text, giving you a stable fingerprint of what was read.

The webpage.ai.v0.1 JSON format

AI2JSON returns a single JSON document per page, using a compact, versioned spec. The contract is designed to stay stable across implementations and upgrades.

Field Type Purpose
spec string Always "webpage.ai.v0.1" for this version.
url string URL requested by the client.
canonical_url string Canonical URL if detected, otherwise mirrors url.
last_fetched string ISO 8601 UTC timestamp of the snapshot.
language string Best-guess language code, e.g. "en" or "fr".
content_hash string SHA-256 of extracted text, prefixed with sha256:.
title string Main page title.
summary string Short factual summary of the content.
type string One of "article", "homepage", or "unknown".
sections[] array Ordered blocks with heading, level, id, text.
links[] array Optional extracted links for navigation or reference.
meta object Best-effort metadata like site name, author, dates, tags.
proof object Reserved for future content integrity proofs.

The formal specification lives in spec/webpage-ai-v0.1.md in the repository and is kept stable across the whole v0.1 lifecycle.

API in one glance

AI2JSON v0.1 exposes a single, stateless endpoint. You send a URL and an API key, you receive a webpage.ai.v0.1 document.

Base URL (example) https://api.ai2json.com Endpoint GET /v1/transform?url=<ENCODED_URL> Headers Authorization: Bearer YOUR_API_KEY Accept: application/json Query parameters url Required. Public page URL to transform. lang Optional. Preferred language for future advanced summaries. no_cache Optional. If true, forces a fresh fetch. Behaviour 200 OK Body is a webpage.ai.v0.1 JSON document. 4xx Client errors (missing url, invalid url, auth, etc.). 5xx Upstream or internal errors, with JSON error payload.

A more detailed documentation (including error codes and edge cases) lives in the repository and will evolve as the API grows from v0.1 to v1.0.

Roadmap

AI2JSON starts small on purpose. The goal is to keep the core contract stable while adding capabilities that matter for real AI workloads.

v0.1 · HTML → JSON

Single endpoint, webpage.ai.v0.1 spec, basic extraction and summary, single API key flow for early users.

v0.2 · Smarter summaries

LLM-powered summaries with explicit language control, improved metadata, better handling of long technical pages.

v0.3 · Dynamic pages

Optional JavaScript rendering for SPAs and dashboards, with the same JSON contract.

v1.0 · Production standard

Dashboards, multi-key management, quotas, monitoring, webhooks, and a fully versioned specification.
