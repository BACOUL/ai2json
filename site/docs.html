API Quickstart (v0.1)

A2J exposes a single stateless endpoint. You send a URL and an API key. You receive a webpage.ai.v0.1 JSON document ready for LLMs, agents, and RAG pipelines.

One endpoint Stable schema Content hash
Base

Base URL

Use this as a placeholder until you deploy your Cloudflare Worker endpoint.

https://api.ai2json.com
Authentication

API Key

Send your key using the Authorization header:

Authorization: Bearer YOUR_API_KEY

If the key is missing or invalid, the API returns 401.

Endpoint

Transform a webpage

GET /v1/transform converts a public URL into a webpage.ai.v0.1 document.

GET /v1/transform?url=<ENCODED_URL>

Example:

curl "https://api.ai2json.com/v1/transform?url=https://example.com/article" \ -H "Authorization: Bearer YOUR_API_KEY" \ -H "Accept: application/json"
Parameters

Query parameters

url is required and must be a public HTTP/HTTPS URL.

url Required. Public page URL to transform. lang Optional. Preferred language for future advanced summaries. no_cache Optional. If true, forces a fresh fetch.
Response

webpage.ai.v0.1

Response body is a single JSON document following the spec. Excerpt:

{ "spec": "webpage.ai.v0.1", "url": "https://example.com/article", "canonical_url": "https://example.com/article", "last_fetched": "2025-01-02T10:11:12Z", "language": "en", "content_hash": "sha256:abc123...", "title": "Example title", "summary": "Short factual summary...", "type": "article", "sections": [ { "heading": "Intro", "level": 1, "id": "intro", "text": "..." } ], "links": [], "meta": {}, "proof": {} }
Errors

Error codes

A2J returns JSON error payloads with a predictable shape.

400 Missing/invalid url 401 Missing/invalid API key 403 Blocked by policy (private IPs, local networks, etc.) 429 Rate limited 5xx Upstream or internal error

Example error body:

{ "error": { "code": "invalid_url", "message": "The 'url' parameter must be a valid public http(s) URL." } }
